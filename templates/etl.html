About the Data

The data originates from a competition sponsored by America Express, a division in India, and Analytics Vidhya that tests contestants each year in different analytical capabilities, last year was on machine learning. This data in-turn presented us a great base for our own machine learning project exploring multiple models, to figure out which one gives the most accurate predictions that aptly reflect the data.

Cleaning Data

Cleaning the data was easy, the Amex Hackathon Competition did a relatively good job at giving the participants a partially cleaned up dataset. They probably left some wiggle room for the contestants to clean it up even more, so their own machine learning methodologies could properly function, hence the competition.

Our team had to do the same. We ran into a few issues that ranged in from minors one to major ones for the ETL portion of this project. There were several small issues such as; incorrect data inputs from several columns that had to be expunged . As well as, we deleted any rows that contained missing data in any of the cells, and dropped unnecessary columns, so it wouldn’t trip much any of our machine learning algorithms.

The major issue was that some of our data was string based, answering questions with a confirmation or a negative answer, and even what type of job the Amex cardholder had. The problem was how to turn those string answers into numeric values, so our machine learning methodologies could properly read the data. Google had numerous complex intricate options to do this, but our TAs helpfully pointed us to the get_dummies function that achieved the conversion to numeric data in an easy manner. However, it did enlarge the dataset as whole quite a bit, increasing the size in-turn slowed down our different methods, but the pros from the get_dummies severely outweighs the con of the increased size and time. We then split the remaining data with sklearn’s train_test_split and viola we are done transforming the data to something workable. All this work ended with a clean csv for all the project members to start their own work in understanding the accuracy and strengths of each of the methods.
